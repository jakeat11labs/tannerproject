What We Practiced

agent workflow · infrastructure

AI coding agents

  • Let Claude Code drive implementation while Jake narrated what mattered.
  • Used planning, commits, verification, and refactors as part of the lesson.
  • Kept the focus on asking the agent for outcomes, not hand-editing every file.

Runtime boundaries

  • Local `.env.local` is for local development.
  • Vercel env vars are for hosted runtime secrets.
  • GitHub should hold code and history, not private credentials.

Vercel Marketplace decision

  • Provisioning Supabase directly would create the database, but leave the app config as a separate manual step.
  • Provisioning Supabase through Vercel links the service to the Vercel project and auto-populates the generated Supabase URL and API keys as environment variables.
  • After that, `vercel env pull` brings the same configuration down for local development without copying secrets by hand.

Production rough edges

  • Adjusted around Vercel Hobby-plan cron limits.
  • Handled BotID placement and protected signed-URL requests.
  • Started the last-mile checklist for webhook and deploy verification.
